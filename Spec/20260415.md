## 18. 邊框標示（Border Mark）

### 18.1 功能概述

在主板（A→B）與摺疊板（B→D）的大面板四邊往內標示出一個較深顏色的填色區域，協助施工時確認安全邊距。

### 18.2 顯示條件

| 開關 ID | 參數鍵 | 預設 |
|---------|--------|------|
| `#togBorderMark` | `showBorderMark` | on |

關閉時，兩個寬度 slider row 一併隱藏（`display: none`）。

### 18.3 邊框寬度參數

| 滑桿 ID | 參數鍵 | 範圍 | 預設 | 作用邊 |
|---------|--------|------|------|--------|
| `#borderMarkWidth` | `borderMarkWidth` | 3.6–10 cm | 5 cm | A 端、D 端、左右 Z 邊 |
| `#borderMarkJunctionWidth` | `borderMarkJunctionWidth` | 3–5 cm | **3 cm** | B 交接端（主板 B 端 + 摺疊板 B 端） |

Step 均為 0.1 cm；值以公尺儲存（`cm / 100`）。

### 18.4 幾何定義

設 `M = borderMarkWidth`，`J = borderMarkJunctionWidth`。

#### 正面視圖（Y-Z 投影，`frontview.js`）

**主板**（大面尺寸 MAIN_LEN × BOARD_W）：

```
內框上邊  mT = AY_w − M × cos(θ)     // A 端往內 M
內框下邊  mB = BY_w + J × cos(θ)     // B 端往內 J
內框左邊  z  = M                      // Z=0 邊往內 M
內框右邊  z  = BOARD_W − M            // Z=BW 邊往內 M
```

**摺疊板**（大面尺寸 FOLD_LEN × BOARD_W）：

```
內框上邊  fT = BY_w + J × fdy         // B 端往內 J（沿折疊方向）
內框下邊  fB = DY_w − M × fdy         // D 端往內 M
內框左邊  z  = M
內框右邊  z  = BOARD_W − M
```

#### 3D 視圖（`draw.js`）

在可見大面（外側面或內側面，由 shoelace 法判斷）繪製外框 minus 內框（evenodd fill）：

- topMarg / botMarg 對應沿板方向兩端各自的邊距（主板：top=M, bot=J；摺疊板：top=J, bot=M）
- Z 方向兩邊固定使用 M

#### 側視圖 2D（`draw.js`，`show3D=false`）

在板端截面繪製填色四邊形：

| 位置 | 寬度 |
|------|------|
| 主板 A 端 | M |
| 主板 B 端 | J |
| 摺疊板 B 端 | J |
| 摺疊板 D 端 | M |

### 18.5 視覺樣式

| 屬性 | 值 |
|------|-----|
| 填色 | `rgba(0,0,0,0.35)`（暗色疊加，使邊框區域呈較深同色調） |
| 繪製方式 | `ctx.fill('evenodd')`（外框路徑 + 內框路徑組合） |
| 深度排序（3D） | 可見面 depth − 0.002，確保繪於板面之上 |
| 線型 | 無邊線，純填色 |

## 19. 正面視圖分割線

### 19.1 功能概述

在正面視圖的主板與摺疊板面板中，以垂直虛線將板寬分割為數個欄位，並在各欄頂端標示欄寬（公尺）；若邊框標示開啟，於每條分割線為中心繪製等寬陰影帶（寬度沿用 `borderMarkWidth`）。

### 19.2 分割配置

| 面板 | 欄寬（左→右，m） | 分割線數 |
|------|-----------------|---------|
| 主板（A→B） | 0.48 · 0.90 · 0.90 · 0.90 · 0.48 | 4 |
| 摺疊板（B→D） | 1.18 · 1.30 · 1.18 | 2 |

兩組分割線各自獨立；分割線 Z 位置為欄寬累積和。

### 19.3 邊框帶

| 屬性 | 值 |
|------|-----|
| 觸發條件 | `showBorderMark = true` |
| 中心 | 分割線 Z 位置 |
| 寬度（m） | `borderMarkWidth`（與 §18 通用邊框寬共用） |
| 高度 | 各面板全高（`yTop → yBot`） |
| 填色 | `rgba(0,0,0,0.35)` |
| 繪製順序 | 陰影帶先繪，虛線後繪（確保線條可見） |

### 19.4 欄寬標籤

- 字體：`FS.dim`，`monospace`
- 位置：各欄水平中心，`yTop + FS.dim·dpr + 6·dpr`（面板頂端內側）
- 顏色：主板 `rgba(196,137,74,0.65)`；摺疊板 `rgba(122,184,232,0.65)`

### 19.5 顯示條件

- 主板分割線：恆顯示（2D 模式下）
- 摺疊板分割線：`showFold = true` 時顯示

## 20. Git Hooks

Hook 腳本置於 `.githooks/`（納入版本控制），並以 `git config core.hooksPath .githooks` 啟用。Clone 後需執行一次此指令。

### 20.1 `pre-commit`（封鎖）

| 規則 | 觸發條件 | 訊息前綴 |
|------|---------|---------|
| R1 | `Code/` 有 staged 檔案，但無任何 `Spec/` staged 檔案 | `[R1]` |
| R2 | `Spec/YYYYMMDD.md` staged，但對應 `Spec/YYYYMMDD.design.md` 未 staged | `[R2]` |
| R3 | 新建 spec 檔案（`--diff-filter=A`）未與其對應檔案同時出現（fn 缺 design 或反之） | `[R3]` |

任一規則命中 → `exit 1` 中止 commit。

### 20.2 分支切換規則

git 無 pre-checkout hook，無法以 hook 封鎖切換。此規則改以 `CLAUDE.md` 強制要求：**`Code/` 有未提交異動時，禁止切換分支；必須先 commit 或 stash。**

## 21. 摺疊板折疊範圍與 2D 視角隱藏規則

### 21.1 折疊角公式

`foldRad = π × (1 − angle/40)`

| 牆面夾角 θ | foldRad | 說明 |
|-----------|---------|------|
| 0° | π（180°） | 完全折回，與主板平行反向 |
| 40° | 0° | 與主板共面（D = C） |

### 21.2 正面視圖隱藏條件

`fdy = sin(foldDir) = −cos(θ + foldRad)`

當 `fdy >= 0`（B→D 朝天花板方向），摺疊板已折回主板背面，**僅正面視圖**隱藏摺疊板相關元素；側視圖不受影響（詳 §21.3）。

臨界角：令 `θ_r + π(1 − θ/40) = π/2`，解得 `θ = 180/7 ≈ 25.7°`。  
`θ < 25.7°` 時 `fdy > 0` → 正面視圖隱藏；`θ ≥ 25.7°` 時 `fdy < 0` → 顯示。

隱藏元素（`frontview.js`，`fdy >= 0` 時）：摺疊板矩形、邊框標示、螺絲孔/格點、分割線、`B→D` 標籤、D 點標籤、尺寸線。

### 21.3 側視圖恆顯示

側視圖（X-Y 投影）中，摺疊板沿 fdx（恆負，朝牆方向）從 B 延伸，永遠不被主板遮擋。`show3D = false` 時無條件繪製所有摺疊板元素：前面（z=0 面）、邊框標示、D 關鍵點、`B→D` 標籤。`show3D = true` 由深度排序自然處理。

## 22. 2D 側視圖牆面與天花板

### 22.1 繪製時機

`show3D = false` 時，在側視圖左半部繪製牆面與天花板結構填色帶，提供空間方位參考。

### 22.2 幾何與樣式

| 元素 | 位置 | 填色 | 邊線 |
|------|------|------|------|
| 牆面帶 | x = 0 左側 8px，y = 0 → CEIL_H | `#2d3548` | `#4a5070`，1.5px，沿 x=0 |
| 天花板帶 | y = CEIL_H 上方 8px，x = 0 → roomMaxX | `#2a2e40` | `#4a5070`，1.5px，沿 y=CEIL_H |

帶寬固定 `8 × dpr` px（與比例尺無關）。

### 22.3 標籤

| 標籤 | 位置 | 對齊 |
|------|------|------|
| 牆面 | `p(0, CEIL_H×0.5)` 左側 10px | `right` |
| 天花板 | `p(roomMaxX×0.5, CEIL_H)` 上方 12px | `center` |

字型同其他 room label（`FS.room`，`#5a5d6e`）。
