## 0. 人工技術決策（最高優先級）

> 本節記錄對使用者不可見的內部實作選擇——資料結構、表示法、演算法模式。這些決策具有最高優先權，**不得在未取得人工確認的情況下更動**。每條決策附有捨棄的替代方案與理由。

---

### ── 幾何計算 ──

### TD-01　`getGeom()` 每幀完整重算，無快取

**決策：** `getGeom()` 為純函式，每次呼叫 `draw()` 時完整重新計算所有幾何點，不快取任何中間結果。

**捨棄的替代方案：** 僅在 `params` 改變時重算並快取（需 dirty flag 或響應式系統）。

**理由：** 幾何計算量極小（數十次三角函式），每幀重算的開銷可忽略；省去快取邏輯降低複雜度，也消除快取失效的 bug 風險。

---

### ── 投影 ──

### TD-05　正面視圖改為固定方向 Y-Z 投影，隨牆面夾角同步更新

> TD-03（折疊段以 `cos(foldRad)` 縮短）與 TD-04（頂部 `fvOY` 對齊）均已被本條取代，不再適用。

**決策：** 正面視圖觀察方向固定為 +X（從室內看向牆面），投影面為 Y-Z 平面。主板可見高度 = `MAIN_LEN × cos(θ)`，折疊段末端 Y = `BY_w + FOLD_LEN × (−cos(θ + foldRad))`，兩者均隨 `params.angle` 變化。

**捨棄的替代方案：** TD-03 的「沿板面法向量觀察」——觀察方向跟著板旋轉，主板永遠顯示為滿高 MAIN_LEN。

**理由：** 固定方向投影可直觀呈現板的「房間中的實際正視影像」，夾角越大板越矮，符合使用者對空間比例的直覺。

---

### TD-06　側視圖與正面視圖共用比例尺 `S`，`autoS` 同時受兩視圖寬度約束

**決策：** 2D 模式下 `autoS = min(sideW/sceneW, frontW/BOARD_W, (H−margin×2)/sceneH)`，其中 `frontW = W/2 − margin − 90·dpr`。

**捨棄的替代方案：** 正面視圖自行計算獨立的 `fvS`。

**理由：** 工程圖慣例要求同一圖紙上各視圖使用相同比例；統一比例尺後，側視圖與正面視圖的距離可直接對比，不需換算。

---

### ── 可見面判斷 ──

### TD-02　螺絲孔可見面以 shoelace 帶號面積判斷，不用法向量點積

**決策：** 可見面判斷使用投影多邊形的帶號面積（`sArea`）；面積 < 0 → 外側面朝向視線。

**捨棄的替代方案：** 計算板面法向量與視線向量的點積（需要 3D 叉積及視線向量）。

**理由：** shoelace 法直接在 canvas 投影座標上操作，無需維護 3D 法向量；實作更短，且對 cabinet oblique 投影同樣正確。

**附：深度排序修正（bug fix）** 螺絲孔的 drawList depth 原取外側面深度，導致 `isOuter=false` 時孔被內側面蓋住。修正為取實際可見面的深度。

---

### ── 板料構造 ──

### TD-15　板材本體厚度（T）與框架總深度（T_FRAME）分離

**決策：** 新增 `T_FRAME = T + 0.027`，A/G/B 鉸接點及所有外側邊點使用 `T_FRAME`；`T` 僅保留做面板厚度識別與雙層標註。

**捨棄的替代方案：** 將 T 直接改為 4.5 cm（不區分面板與框架）。

**理由：** 面板（1.8 cm）與框架凸出（2.7 cm）對應不同構件，分離後可各自用於尺寸標註與未來渲染，且公式直觀（`T_FRAME = T + 框凸出量`）。

---

### TD-16　面板在前、邊框在後構造確認

**決策：** 面板（T = 1.8cm）覆蓋外側面（朝房間），全面完整拼接；邊框（2.7cm）固定於板面後方（內側）。所有截面分界線、標註、資訊面板均以此為準。

**捨棄的替代方案：** 傳統「框在外、板嵌入內」構造（邊框於正面可見）。

**理由：** 使用者確認此板件為「板面後方裝框架」設計；正面視圖不應顯示邊框輪廓，分割線代表面板拼接縫而非邊框邊緣。

---

### ── 渲染 ──

### TD-08　邊框標示採 evenodd 填色，非虛線描邊

**決策：** 邊框標示以兩條閉合路徑（外框 + 內框）組合後呼叫 `ctx.fill('evenodd')`，填色為 `rgba(0,0,0,0.35)` 暗色疊加。

**捨棄的替代方案：** 描繪內框虛線（`setLineDash + stroke`）。

**理由：** 填色區域能明確呈現需要避開的空間，且在不同底色（木板棕、折疊藍）上均不需調整對比色。evenodd 規則利用路徑繞行方向自然形成框形，無需計算四個獨立矩形。

---

### TD-09　B 交接端寬度獨立參數，不共用通用邊框寬

**決策：** 新增 `borderMarkJunctionWidth`（3–5 cm）獨立控制主板 B 端與折疊板 B 端的邊框寬度；通用 `borderMarkWidth` 僅作用於 A 端、D 端及左右 Z 邊。

**捨棄的替代方案：** 所有四邊共用同一寬度。

**理由：** B 交接處兩側均有邊框，若與 A/D 端同寬，視覺上容易產生誤導感；獨立參數讓施工人員可針對交接處設定更嚴格（或更寬鬆）的安全邊距。

---

### TD-10　分割線陰影帶共用 `borderMarkWidth`，不設獨立參數

**決策：** 分割線邊框帶的寬度直接沿用 `borderMarkWidth`，不新增獨立滑桿。

**捨棄的替代方案：** 新增 `divisionMarkWidth` 參數。

**理由：** 邊框帶的語意為「以分割線為中心的施工安全邊距」，與板緣邊框概念一致，寬度應相同。共用參數減少 UI 複雜度。

---

### TD-12　2D 正面視圖摺疊板折回背面時不繪製（側視圖恆顯示）

**決策：** `fdy >= 0` 時，**僅** `frontview.js` 跳過摺疊板繪製；`draw.js` 側視圖無條件繪製所有摺疊板元素。以 `const foldVisible = params.showFold && fdy < 0` 統一控制正面視圖；`draw.js` 移除所有 `fdy < 0` 條件判斷。

**捨棄的替代方案：** 兩個視圖共用相同的隱藏條件。

**理由：** 側視圖（X-Y 投影）中，fdx 恆為負（恆朝牆），永不被主板遮擋。正面視圖（Y-Z 投影）中，折回的摺疊板 Y 範圍與主板重疊，隱藏可避免視覺混淆。

---

### TD-13　折疊角範圍改為 0°–180°

**決策：** `foldRad = π × (1 − angle/40)`，使 θ=0° 時摺疊板與主板完全平行反向（180°），θ=40° 時共面（0°）。

**捨棄的替代方案：** 原始值 `(2π/3) × (1 − angle/40)`（最大 120°）。

**理由：** 180° 對應實物完全收合的物理極限，讓使用者能模擬板子貼牆收起的狀態；120° 無法表達「完全折疊」。

---

### TD-14　2D 側視圖以固定像素帶表示牆面與天花板

**決策：** 在 `show3D = false` 時，沿 x=0（牆）與 y=CEIL_H（天花板）繪製 8×dpr px 固定寬度的填色帶，並在帶邊緣畫 1.5px 邊線。

**捨棄的替代方案：** 以真實厚度（公尺）換算像素（隨縮放變化，在小尺度下可能細到不可見）。

**理由：** 固定像素帶在任何縮放比例下均保持清晰可見，且與技術圖慣例一致。帶寬作用為方位指示而非尺寸表達，不需與實際牆厚比例準確。

---

### TD-17　3D 視圖分割帶以填色矩形渲染，不使用虛線

**決策：** 在 3D 可見大面上，以 `rgba(0,0,0,0.35)` 填色矩形表示分割帶（Z 範圍 `±MARGIN/2`，全板長），與外框邊框標示同層（depth − 0.002）。

**捨棄的替代方案：** 在 3D 面上繪製虛線。

**理由：** 3D 視圖中虛線在斜平行投影下難以辨認方向；填色帶直接呈現「框架材料佔據的寬度」，與外框標示保持一致的視覺語言。分割線位置由 `MAIN_DIV_WIDTHS`/`FOLD_DIV_WIDTHS` 常數驅動，兩視圖共用一來源，避免不一致。

---

### TD-18　主板中段水平框架渲染

**決策：** 填色帶（`showBorderMark`）+ 水平虛線（中心，同縱向分割線色與 dash 樣式）出現於三個視圖；`showDims` 時在正面視圖板左側加垂直標註線（A 端至框架中心，cm）。位置計算：中點 `u_nominal` 起，以 0.5 cm 步進向 A 端避開 `SCREW_U`/`CELL_U` 孔行（閾值 `frameH/2 + HOLE_R_M`）。

**捨棄的替代方案：** 固定位置不避孔；中心線用實線；尺寸標在側視圖。

**理由：** 步進避孔確保 `borderMarkWidth` slider 任意值下均不穿孔。水平虛線沿用縱向分割線慣例保持視覺一致。尺寸標左側與主板高度標（右側）不重疊。

---

### ── 標註 ──

### TD-19　B 點厚度標註拆分為板厚（外側）與框深（內側）

**決策：** 保留 Bw → Bo 全長線段；在 `Bw + (T_FRAME-T) × 法向量` 處加重線段至 Bo，標示面板層（1.8cm）；Bw 端標「框 2.7cm」，Bo 端標「板 1.8cm」。另加面板分界虛線貫穿全板長。

**捨棄的替代方案：** 刻度位於 `Bw + T × 法向量`（從內側量起 1.8cm，與實際構造相反）。

**理由：** 面板位於外側面（朝房間），邊框位於板面後方；刻度應從外側量起反映實物施工位置。

---

### ── 程式架構 ──

### TD-07　`board_viz.js` 拆分為 ES Modules，以 `type="module"` 載入

**決策：** 將單一檔案依職責拆分為六個模組（`constants.js`、`geometry.js`、`state.js`、`frontview.js`、`draw.js`、`events.js`），`board_viz.js` 成為入口點。兩個 HTML 檔案改為 `<script type="module">`。

**捨棄的替代方案：** 多個有序 `<script>` 標籤（共用全域變數，無 import/export）。

**理由：** ES Modules 提供明確的依賴邊界，消除隱性全域變數耦合；可變狀態（`params`、`view`）改以物件屬性傳遞，各模組皆可讀寫同一參考。

---

### TD-11　Git hooks 置於 `.githooks/` 納入版本控制，以 `core.hooksPath` 啟用

**決策：** Hook 腳本置於版本控制目錄 `.githooks/`（僅 `pre-commit`），以 `git config core.hooksPath .githooks` 啟用。分支切換規則改由 `CLAUDE.md` 強制要求。不使用 Husky 等工具。

**捨棄的替代方案：** 直接寫入 `.git/hooks/`（不可版本控制）；Husky（引入 npm 依賴，違反無 build system 原則）。

**理由：** `.githooks/` 方案在無依賴約束下成本最低；`CLAUDE.md` 讓 AI agent 在執行前就知道限制，比 post-checkout 事後警告更有效。

**理由：** 該頁面未從任何其他頁面連結，屬孤立檔案，移除以保持目錄整潔。

## 5. 計算與渲染管線

### 5.1 `getGeom()` 輸出結構

```
Aw, Ao, Ac          — A 點（靠牆側、外側、中心線）
Bw, Bo, Bc, Bfo     — B 點（靠牆側、外側、中心線、摺疊外側）
Cw, Co              — C 點（虛擬延伸端，靠牆側/外側）
Dw, Do              — D 點（摺疊端，靠牆側/外側）
boardDir, foldDir   — 主板/摺疊段方向角（rad）
fdx, fdy, fnx, fny  — 摺疊段單位向量與法向量
Gw                  — G 點座標（靠牆側）
HG                  — |H→G| 距離
LEN_HM              — 剛性桿半長（= 0.5·|H→G|@θ=0）
M                   — M 點座標（兩圓交點，彎向 A 側）
MA                  — |M→A| 距離
refFloorX           — 地面內縮參考點 X
projOnBoardY        — 內縮點對主板外側面的投影高度（null 若不相交）
```

### 5.2 `draw()` 執行順序

1. **清除畫布**，填底色 `#0e0f12`
2. **投影常數計算**：`OBL_COS`, `OBL_SIN`, `cosV`, `sinV`, `S`, `OX`, `OY`
3. **定義投影函式** `p(x,y,z)→[canvas_x, canvas_y]`（見 §6）
4. **2D 側視圖牆面與天花板**（`show3D=false` 時，見功能規格 §21）
5. **室內環境**（`show3D` 時）：牆面、天花板、地面虛線
6. **地面前緣線**（常顯示）
7. **建立 `drawList[]`**：
   - 7a. 主板螺絲孔 entry（`show3D && showScrewHoles`）：depth = 可見面 depth − 0.001
   - 7b. 折疊段螺絲孔 entry（`show3D && showScrewHoles && showFold`）：同上
   - 7c. 主板邊框標示 entry（`showBorderMark`）：depth = 可見面 depth − 0.002
   - 7d. 折疊板邊框標示 entry（`showBorderMark && showFold`）：同上
   - 7e. 分割帶 entries（`showBorderMark`）：主板 4 條 + 摺疊板 2 條（`show3D` 時）
   - 7f. 中段水平框架 entry（`showBorderMark`）
   - 7g. boardFaces（主板 + 摺疊段各面）、中心線、Ghost C
8. **加入剛性桿**（`showQuarter`）：H→G 灰虛線；3D 模式加深度連接線；遠組/近組桿加入 drawList
9. **排序 `drawList` 由深到淺**，依序執行 `drawFn`
10. **剛性桿長度標籤**（H-M、M-G 中點，僅近組）
11. **H→G 總距離標籤**、**M→A 距離虛線標籤**
12. **錨點**：H（菱形綠）、I（菱形橙）
13. **G、M 點標記**（`showQuarter`）
14. **關鍵點** A/B/C/D（空心圓）
15. **標註尺寸**（`showDims`）：投影虛線、尺寸線、角度弧、內縮高度、B 點雙層厚度標註
16. **室內文字標籤**：牆面、天花板、地面
17. **板段標籤**：`A→B 2.40m`、`B→D 0.65m`
18. **正面視圖**（`show3D=false`）：調用 `drawFrontView()`

## 6. 投影函式

```js
const p = (x, y, z=0) => {
  const h = x*cosV + z*sinV;         // 水平分量
  const d = -x*sinV + z*cosV;        // 深度分量
  return [
    OX + h*S + d*S*OBL_COS,
    OY - y*S - d*S*OBL_SIN
  ];
};
```

`OX` 以場景 XZ 中心為旋轉基準，確保旋轉時畫面不左右偏移（見功能規格 §9.3）。
