# 木板構型視覺化工具 — 設計規格文件

> 版本：4.5 | 對應功能規格：v4.5（`Spec/20260512.md`）| 更新：2026-05-12

---

## 0. 人工技術決策（最高優先級）

> 本節記錄對使用者不可見的內部實作選擇——資料結構、表示法、演算法模式。這些決策具有最高優先權，**不得在未取得人工確認的情況下更動**。

---

### ── 幾何計算 ──

### TD-01　`getGeom()` 每幀完整重算，無快取

**決策：** `getGeom()` 為純函式，每次呼叫 `draw()` 時完整重新計算所有幾何點，不快取任何中間結果。

**捨棄的替代方案：** 僅在 `params` 改變時重算並快取（需 dirty flag 或響應式系統）。

**理由：** 幾何計算量極小（數十次三角函式），每幀重算的開銷可忽略；省去快取邏輯降低複雜度。

---

### TD-21　A 點沿天花板滑動，Cw.x 恆為 0

**決策：** `Aw = { x: BOARD_LEN × sin θ, y: CEIL_H }`，使底端 Cw.x ≡ 0 在所有 θ 下成立。H 點固定於 (0, CEIL_H)；G 點為主板上固定位置（`Gw = Aw + gOffset × dir`），隨板移動。

**捨棄的替代方案：** A 點固定、H 點追蹤 A 的高度移動（v4.4 以前的 H-M-G 剛性桿模型）。

**理由：** 物理直覺：板的底端（C）始終貼牆，因此 C.x = 0 是自然的幾何約束，導出 A.x = BOARD_LEN × sin θ。剛性桿系統移除後，此滑動模型是唯一在所有角度下自洽的幾何描述。

---

### TD-22　移除 H-M-G 剛性桿系統

**決策：** 完全移除 M 點、LEN_HM、M→A 距離及所有剛性桿渲染（`addRod` 呼叫）。`showQuarter` 開關改為控制「G 定位點」顯示（G 點 + H→G 灰虛線）。

**捨棄的替代方案：** 保留剛性桿作為輔助量測工具。

**理由：** 使用者明確指定移除剛性桿；新的幾何模型（A 滑動）使剛性桿失去物理意義。

---

### TD-23　折疊角公式改為 `π × (40 − θ) / 35`

**決策：** `foldRad = π × (40 − θ) / 35`，精確達到 θ=40° 時 0°、θ=5° 時 180°。

**捨棄的替代方案：** 舊公式 `π × (1 − θ/40)`（θ=5° 時達 157.5°，非 180°）；`π × (1 − θ/40)` 的另一修正 `π × (40 − θ) / 40`（θ=0° 時 180°，但 slider 最小值為 5°）。

**理由：** 新公式以 slider 實際範圍 [5°, 40°] 為映射域，確保兩端點精確對應 0° 和 180°，不依賴 slider 範圍外的外推。

---

### ── 投影 ──

### TD-05　正面視圖固定方向 Y-Z 投影，隨牆面夾角同步更新

**決策：** 正面視圖觀察方向固定為 +X，投影面為 Y-Z 平面。主板可見高度 = `MAIN_LEN × cos(θ)`，折疊段末端 Y = `BY_w + FOLD_LEN × sin(foldDir)`，兩者均隨 `params.angle` 變化。

**捨棄的替代方案：** 觀察方向跟著板旋轉（TD-03，已作廢）。

**理由：** 固定方向投影呈現板在房間中的實際正視影像，夾角越大板越矮，符合使用者直覺。

---

### TD-06　側視圖與正面視圖共用比例尺 `S`

**決策：** `autoS = min(sideW/sceneW, frontW/BOARD_W, (H−margin×2)/sceneH)`，`frontW = W/2 − margin − 90·dpr`。

**捨棄的替代方案：** 正面視圖自行計算獨立的 `fvS`。

**理由：** 工程圖慣例要求同一圖紙上各視圖使用相同比例。

---

### ── 可見面判斷 ──

### TD-02　螺絲孔可見面以 shoelace 帶號面積判斷

**決策：** 可見面判斷使用投影多邊形的帶號面積（`sArea`）；面積 < 0 → 外側面朝向視線。

**捨棄的替代方案：** 3D 法向量點積。

**理由：** shoelace 法直接在 canvas 投影座標上操作，對 cabinet oblique 投影同樣正確，實作更短。

---

### ── 板料構造 ──

### TD-15　T 與 T_FRAME 分離

**決策：** `T_FRAME = T + 0.027`；A/G/B 鉸接點及所有外側邊點使用 `T_FRAME`；`T` 保留做面板厚度識別。

**捨棄的替代方案：** 將 T 直接改為 4.5 cm。

**理由：** 面板（1.8 cm）與框架凸出（2.7 cm）對應不同構件，分離後可各自用於標註。

---

### TD-16　面板在前、邊框在後構造確認

**決策：** 面板（T）覆蓋外側面（朝房間）；邊框（2.7 cm）固定於板面後方（內側）。

**捨棄的替代方案：** 框在外、板嵌入內。

**理由：** 使用者確認此板件為「板面後方裝框架」設計。

---

### ── 渲染 ──

### TD-08　邊框標示採 evenodd 填色

**決策：** 兩條閉合路徑（外框 + 內框）呼叫 `ctx.fill('evenodd')`，填色 `rgba(0,0,0,0.35)`。

**捨棄的替代方案：** 虛線描邊。

**理由：** 填色區域在不同底色上均不需調整對比色；evenodd 自然形成框形，無需計算四個獨立矩形。

---

### TD-09　B 交接端寬度獨立參數

**決策：** `borderMarkJunctionWidth`（3–5 cm）獨立控制 B 交接端；通用 `borderMarkWidth` 作用於其他三邊。

**捨棄的替代方案：** 所有四邊共用同一寬度。

**理由：** B 交接處兩側均有邊框，需獨立設定。

---

### TD-10　分割線陰影帶共用 `borderMarkWidth`

**決策：** 分割線邊框帶的寬度直接沿用 `borderMarkWidth`，不設獨立滑桿。

**理由：** 邊框帶語意一致，共用參數減少 UI 複雜度。

---

### TD-12　2D 正面視圖摺疊板折回背面時不繪製（側視圖恆顯示）

**決策：** `fdy >= 0` 時，**僅** `frontview.js` 跳過摺疊板繪製；`draw.js` 側視圖無條件繪製。以 `const foldVisible = params.showFold && fdy < 0` 統一控制。

**捨棄的替代方案：** 兩個視圖共用相同隱藏條件。

**理由：** 側視圖（X-Y）中 fdx 恆負，永不被主板遮擋；正面視圖中折回的摺疊板與主板重疊，隱藏避免視覺混淆。

---

### TD-14　2D 側視圖以固定像素帶表示牆面與天花板

**決策：** `show3D = false` 時，沿 x=0（牆）與 y=CEIL_H（天花板）繪製 8×dpr px 固定寬度填色帶。

**捨棄的替代方案：** 以真實厚度換算像素。

**理由：** 固定像素帶在任何縮放下均清晰可見，作為方位指示不需比例準確。

---

### TD-17　3D 視圖分割帶以填色矩形渲染

**決策：** 以 `rgba(0,0,0,0.35)` 填色矩形表示分割帶，depth − 0.002。

**捨棄的替代方案：** 在 3D 面上繪製虛線。

**理由：** 3D 斜平行投影中虛線難以辨認；填色帶與外框標示保持視覺一致。

---

### TD-18　主板中段水平框架渲染

**決策：** 填色帶（`showBorderMark`）+ 水平虛線（中心）出現於三個視圖；`showDims` 時在正面視圖板左側加垂直標註線。位置計算：中點起，以 0.5 cm 步進向 A 端避孔。

**捨棄的替代方案：** 固定位置不避孔；中心線用實線。

**理由：** 步進避孔確保任意 `borderMarkWidth` 下不穿孔。

---

### TD-19　B 點厚度標註拆分為板厚（外側）與框深（內側）

**決策：** 保留 Bw → Bo 全長線段；在 `Bw + (T_FRAME-T) × norm` 處加重線段至 Bo；Bw 端標「框 2.7cm」，Bo 端標「板 1.8cm」。另加面板分界虛線貫穿全板長。

**捨棄的替代方案：** 刻度從內側量起（與實際構造相反）。

**理由：** 面板位於外側面，邊框在板面後方；刻度從外側量起反映實物施工位置。

---

### TD-07　ES Modules 拆分

**決策：** 依職責拆分為六個模組，`board_viz.js` 為入口點，`<script type="module">` 載入。

**捨棄的替代方案：** 多個有序 `<script>` 標籤。

**理由：** ES Modules 提供明確依賴邊界，消除隱性全域變數耦合。

---

### TD-11　Git hooks 置於 `.githooks/` 納入版本控制

**決策：** Hook 腳本置於 `.githooks/`，以 `git config core.hooksPath .githooks` 啟用。

**捨棄的替代方案：** 直接寫入 `.git/hooks/`；Husky。

**理由：** 成本最低且可版本控制；`CLAUDE.md` 讓 AI agent 在執行前就知道限制。

---

### TD-20　移除裁切計畫獨立頁面（`cutting_plan.html`）

**決策：** 刪除 `Code/cutting_plan.html`。

**理由：** 該頁面未從任何其他頁面連結，屬孤立檔案。

## 5. 計算與渲染管線

### 5.1 `getGeom()` 輸出結構

```
Aw, Ao, Ac          — A 點（靠牆側、外側、中心線）
Bw, Bo, Bc, Bfo     — B 點（靠牆側、外側、中心線、摺疊外側）
Cw, Co              — C 點（靠牆側/外側）
Dw, Do              — D 點（摺疊端，靠牆側/外側）
boardDir, foldDir   — 主板/摺疊段方向角（rad）
fdx, fdy, fnx, fny  — 摺疊段單位向量與法向量
ddx, ddy            — 主板單位向量（= dir）
GX, GY              — G 點座標（靠牆側）
HX, HY              — H 點座標（= 0, CEIL_H，固定）
HG                  — |H→G| 距離
refFloorX           — 地面內縮參考點 X
projOnBoardY        — 內縮點對主板外側面的投影高度（null 若不相交）
```

### 5.2 `draw()` 執行順序

1. 清除畫布，填底色 `#0e0f12`
2. 投影常數計算（`OBL_COS`、`OBL_SIN`、`cosV`、`sinV`、`S`、`OX`、`OY`）
3. 定義投影函式 `p(x,y,z)`
4. 2D 側視圖牆面與天花板（`show3D=false`）
5. 室內環境（`show3D`）：牆面、天花板、地面虛線
6. 地面前緣線（常顯示）
7. 建立 `drawList[]`：螺絲孔、邊框標示、分割帶、中段框架、板面、中心線
8. G 點（`showQuarter`）：H→G 灰虛線加入繪製；G 點 entry 加入 drawList
9. 排序 `drawList` 由深到淺，依序執行 `drawFn`
10. H→G 距離標籤（`showQuarter`）
11. 錨點：H（菱形綠）、I（菱形橙）
12. G 點標記（`showQuarterPt`，`showQuarter`）
13. 關鍵點 A/B/C/D（空心圓）
14. 標註尺寸（`showDims`）：投影虛線、尺寸線、角度弧、內縮高度、B 點雙層厚度標註
15. 室內文字標籤：牆面、天花板、地面
16. 板段標籤：`A→B 2.40m`、`B→D 0.65m`
17. 正面視圖（`show3D=false`）：`drawFrontView()`

## 6. 投影函式

```js
const p = (x, y, z=0) => {
  const h = x*cosV + z*sinV;
  const d = -x*sinV + z*cosV;
  return [
    OX + h*S + d*S*OBL_COS,
    OY - y*S - d*S*OBL_SIN
  ];
};
```

## 8. 點符號視覺規格

| 點 A | 空心圓 r=8dpr，填色 +44 透明 | #e8c87a |
| 點 B | 同上 | #7ab8e8 |
| 點 C | 同上 | #e87a7a |
| 點 D | 同上 | #b07ae8 |
| 點 H | 菱形 r=7dpr | #7ae8b0 |
| 點 I | 菱形 r=7dpr | #e8a87a |
