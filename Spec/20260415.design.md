## 0. 人工技術決策（最高優先級）

承 v1.3。新增 TD-08。

### TD-01 ～ TD-07　同 v1.3，不重複。

### TD-08　邊框標示採 evenodd 填色，非虛線描邊

**決策：** 邊框標示以兩條閉合路徑（外框 + 內框）組合後呼叫 `ctx.fill('evenodd')`，填色為 `rgba(0,0,0,0.35)` 暗色疊加。

**捨棄的替代方案：** 描繪內框虛線（`setLineDash + stroke`）。

**理由：** 虛線在視覺上為輪廓提示，無法直觀傳達「安全邊距區域」的概念；填色區域能明確呈現需要避開的空間，且在不同底色（木板棕、折疊藍）上均不需調整對比色。evenodd 規則利用路徑繞行方向自然形成框形，無需計算四個獨立矩形。

---

### TD-09　B 交接端寬度獨立參數，不共用通用邊框寬

**決策：** 新增 `borderMarkJunctionWidth`（3–5 cm）獨立控制主板 B 端與摺疊板 B 端的邊框寬度，而通用 `borderMarkWidth` 僅作用於 A 端、D 端及左右 Z 邊。

**捨棄的替代方案：** 所有四邊共用同一寬度。

**理由：** B 交接處兩側均有邊框（主板 bot + 折疊板 top），若與 A/D 端同寬，視覺上兩個邊框靠近時容易產生過窄或過寬的誤導感；獨立參數讓施工人員可針對交接處設定更嚴格（或更寬鬆）的安全邊距，不影響其他三邊。

## 4.2 可變狀態（補充）

```js
params.showBorderMark         = true   // 邊框標示開關
params.borderMarkWidth        = 0.05   // 通用邊框寬（m）
params.borderMarkJunctionWidth = 0.05  // B 交接端邊框寬（m）
```

## 5.2 `draw()` 執行順序（補充）

螺絲孔 entry 之後、ghost C 之前，新增：

**7c. 加入主板邊框標示 entry**（`showBorderMark` 時）：
- 可見面判斷同 TD-02（shoelace 帶號面積）
- depth = faceDepth(可見面) − 0.002（比螺絲孔 −0.001 更前）
- 3D：`drawBoardBorder(mTop, mBot, ddx, ddy, MARGIN, MARG_J)`
- 2D：在板端繪製填色四邊形，寬度 = MARGIN（A 端）或 MARG_J（B 端）

**7d. 加入折疊板邊框標示 entry**（`showBorderMark && showFold` 時）：
- depth = faceDepth(可見面) − 0.002
- 3D：`drawBoardBorder(fTop, fBot, fdx, fdy, MARG_J, MARGIN)`
- 2D：寬度 = MARG_J（B 端）或 MARGIN（D 端）
