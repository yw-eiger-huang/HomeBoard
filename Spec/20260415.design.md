### TD-08　邊框標示採 evenodd 填色，非虛線描邊

**決策：** 邊框標示以兩條閉合路徑（外框 + 內框）組合後呼叫 `ctx.fill('evenodd')`，填色為 `rgba(0,0,0,0.35)` 暗色疊加。

**捨棄的替代方案：** 描繪內框虛線（`setLineDash + stroke`）。

**理由：** 虛線在視覺上為輪廓提示，無法直觀傳達「安全邊距區域」的概念；填色區域能明確呈現需要避開的空間，且在不同底色（木板棕、折疊藍）上均不需調整對比色。evenodd 規則利用路徑繞行方向自然形成框形，無需計算四個獨立矩形。

### TD-09　B 交接端寬度獨立參數，不共用通用邊框寬

**決策：** 新增 `borderMarkJunctionWidth`（3–5 cm）獨立控制主板 B 端與摺疊板 B 端的邊框寬度，而通用 `borderMarkWidth` 僅作用於 A 端、D 端及左右 Z 邊。

**捨棄的替代方案：** 所有四邊共用同一寬度。

**理由：** B 交接處兩側均有邊框（主板 bot + 折疊板 top），若與 A/D 端同寬，視覺上兩個邊框靠近時容易產生過窄或過寬的誤導感；獨立參數讓施工人員可針對交接處設定更嚴格（或更寬鬆）的安全邊距，不影響其他三邊。

### TD-10　分割線陰影帶共用 `borderMarkWidth`，不設獨立參數

**決策：** 分割線邊框帶的寬度直接沿用 `borderMarkWidth`（§18 通用邊框寬），不新增獨立滑桿。

**捨棄的替代方案：** 新增 `divisionMarkWidth` 參數，允許分割線邊框帶與板緣邊框寬度分開設定。

**理由：** 邊框帶的語意為「以分割線為中心的施工安全邊距」，與板緣邊框概念一致，寬度應相同。共用參數減少 UI 複雜度；若日後需獨立，可再行拆分。

---

### TD-11　Git hooks 置於 `.githooks/` 納入版本控制，以 `core.hooksPath` 啟用

**決策：** Hook 腳本置於版本控制目錄 `.githooks/`（僅 `pre-commit`），以 `git config core.hooksPath .githooks` 啟用。分支切換規則改由 `CLAUDE.md` 強制要求，不設 post-checkout hook。不使用 Husky 等工具。

**捨棄的替代方案：** 直接寫入 `.git/hooks/`（不可版本控制）；post-checkout hook 警告（事後提示，無法封鎖，實際效果有限）；Husky（引入 npm 依賴，違反無 build system 原則）。

**理由：** post-checkout 在切換完成後才執行，無法阻止動作；將規則放入 `CLAUDE.md` 讓 AI agent 在執行前就知道限制，比事後警告更有效。`.githooks/` 方案在無依賴約束下成本最低；clone 後執行一次 `git config` 即可。

---

### TD-12　2D 視角摺疊板折回背面時不繪製

**決策：** 計算 `fdy = −cos(θ + foldRad)`；當 `fdy >= 0`（摺疊板朝天花板方向，即已折回主板背面），在 `show3D = false` 時跳過所有摺疊板相關繪製。`show3D = true` 時由深度排序自然處理，不受影響。

**實作方式：** `frontview.js` 以 `const foldVisible = params.showFold && fdy < 0` 統一控制；`draw.js` 在折疊板 2D 前面、D 標籤、B→D 標籤加入 `params.show3D || g.fdy < 0` 條件。

**捨棄的替代方案：** 以 `foldRad > π/2` 判斷（純角度條件，但未考量主板旋轉角 θ 的影響）；以 `Dw.x < Bw.x` 判斷（X 位置比較，與視角方向耦合）。

**理由：** 當摺疊板折入主板與牆壁之間時，在 2D 側視圖中繪製會造成視覺混淆（板形重疊）；隱藏可讓圖面清晰表達「板已收起」的狀態。`fdy` 直接對應正面視圖的高度分量，是最自然的判斷依據。

## 10. 裁切計畫

板材規格：**244 × 122 cm**，鋸縫寬 **0.5 cm**。共需 **5 片板材**。

### 料件清單

| 料件 | 寬（Z） | 長 | 數量 |
|------|--------|----|------|
| 主板 M1、M5 | 48 cm | 239.8 cm | 各 1 |
| 主板 M2、M3、M4 | 90 cm | 239.8 cm | 各 1 |
| 摺疊板 F1、F3 | 118 cm | 65 cm | 各 1 |
| 摺疊板 F2 | 130 cm | 65 cm | 1 |

主板料件長 239.8 cm，須沿 244 cm 軸排放；摺疊板 F2 寬 130 cm > 122 cm，同須沿 244 cm 軸排放。兩 90 cm 料件並排超寬（90 + 0.5 + 90 = 180.5 > 122），故各占一片。主板與摺疊板在 244 cm 方向上無法共板（主板切後僅剩 3.7 cm）；M1、M5（48 cm）改與摺疊板拼入剩餘 57 cm 空間（65 + 0.5 + 48 = 113.5 ≤ 122）。
