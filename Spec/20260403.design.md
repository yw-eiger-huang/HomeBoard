## 0. 人工技術決策（最高優先級）

> 本節記錄對使用者不可見的內部實作選擇——資料結構、表示法、演算法模式。這些決策具有最高優先權，**不得在未取得人工確認的情況下更動**。每條決策附有捨棄的替代方案與理由，以便日後判斷邊界情況。

---

### TD-01　`getGeom()` 每幀完整重算，無快取

**決策：** `getGeom()` 為純函式，每次呼叫 `draw()` 時完整重新計算所有幾何點，不快取任何中間結果。

**捨棄的替代方案：** 僅在 `params` 改變時重算並快取（需 dirty flag 或響應式系統）。

**理由：** 幾何計算量極小（數十次三角函式），每幀重算的開銷可忽略；省去快取邏輯降低複雜度，也消除快取失效的 bug 風險。

`.toggle` 為自訂 CSS 開關；`.toggle.on` 背景變 `--accent`，偽元素 `::after`（白色圓點）向右位移 16px，動畫時間 0.2s。

| `.yellow` | `--accent` | 物理常數 |

| `.green` | `--accent3` | 量測結果 |

## 5. 計算與渲染管線

### 5.1 `getGeom()` 輸出結構

```
Aw, Ao, Ac          — A 點（牆側、外側、中心線）
Bw, Bo, Bc, Bfo     — B 點（牆側、外側、中心線、摺疊外側）
Cw, Co              — C 點（虛擬延伸端，牆側/外側）
Dw, Do              — D 點（摺疊端，牆側/外側）
boardDir, foldDir   — 主板/摺疊段方向角（rad）
fdx, fdy, fnx, fny  — 摺疊段單位向量與法向量
GX, GY              — G 點座標
HG                  — |H→G| 距離
LEN_HM              — 剛性桿半長（= 0.5·|H→G|@θ=0）
MX, MY              — M 點座標（兩圓交點，彎向 A 側）
MA                  — |M→A| 距離
refFloorX           — 地面內縮參考點 X
projOnBoardY        — 內縮點對主板外側面的投影高度（null 若不相交）
```

### 5.2 `draw()` 執行順序

1. **清除畫布**，填底色 `#0e0f12`
2. **投影常數計算**：`OBL_COS`, `OBL_SIN`, `cosV`, `sinV`, `S`, `OX`, `OY`
3. **定義投影函式** `p(x,y,z)→[canvas_x, canvas_y]`（見功能規格 §6.2）
4. **室內環境**（`show3D` 時）：牆面、天花板、地面虛線
5. **地面前緣線**（常顯示）
6. **建立 `boardFaces[]`**：收集所有板面（主板 5+1 面、摺疊段 5+1 面，視模式）
7. **建立 `drawList[]`**：`boardFaces` + 中心線 + Ghost C + 剛性桿（若 `showQuarter`）
   - 每個 entry：`{ depth: float, drawFn: () => void }`
   - `depth = mean(-x·sinV + z·cosV)`（越大越遠）
8. **加入剛性桿**（`showQuarter` 時）：
   - 先畫 H→G 灰虛線
   - 3D 模式：加入深度連接線（立即繪製，不入 drawList）
   - 加入遠組桿（`rodZFar`）和近組桿（`rodZNear`）至 drawList
9. **排序 `drawList` 由深到淺**（Painter's Algorithm），依序執行 `drawFn`
10. **剛性桿長度標籤**（H-M、M-G 中點，僅近組）
11. **H→G 總距離標籤**、**M→A 距離虛線標籤**
12. **錨點**：H（菱形綠）、I（菱形橙）
13. **G、M 點標記**（`showQuarter`）
14. **關鍵點** A/B/C/D（空心圓）
15. **標註尺寸**（`showDims`）：投影虛線、尺寸線、角度弧、內縮高度
16. **室內文字標籤**：牆面、天花板、地面
17. **板段標籤**：`A→B 2.40m`、`B→D 0.65m`

## 6. 投影函式

```js
const p = (x, y, z=0) => {
  const h = x*cosV + z*sinV;         // 水平分量
  const d = -x*sinV + z*cosV;        // 深度分量
  return [
    OX + h*S + d*S*OBL_COS,
    OY - y*S - d*S*OBL_SIN
  ];
};
```

`OX` 以場景 XZ 中心為旋轉基準（v3.4），確保旋轉時畫面不左右偏移（詳見功能規格 §6.3）。

## 7. 互動事件

| 事件 | 行為 |
|------|------|
| 滑桿 `input` | 更新 `params.*`，呼叫 `draw()` |
| Toggle `click` | 切換 `params.*`；3D Toggle 另處理視角滑桿顯示/隱藏及 `savedHViewAngle` |
| `mousedown/move/up/leave` | 平移（`viewOX/Y`） |
| `wheel`（passive:false） | 縮放（`viewScale` 0.15–12，係數 ×1.1 / ×0.9） |
| `touchstart/move/end`（passive:false）| 單指平移、雙指捏放縮放 |
| `window resize` | 呼叫 `resize()`，重設 HiDPI 畫布尺寸 |
