### TD-02　螺絲孔可見面以 shoelace 帶號面積判斷，不用法向量點積

**決策：** 可見面判斷使用投影多邊形的帶號面積（`sArea`）；面積 < 0 → 外側面朝向視線。

**捨棄的替代方案：** 計算板面法向量與視線向量的點積（需要 3D 叉積及視線向量）。

**理由：** shoelace 法直接在 canvas 投影座標上操作，無需維護 3D 法向量；實作更短，且對 cabinet oblique 投影同樣正確。

**附：深度排序修正（v3.5 bug fix）** 螺絲孔的 drawList depth 原來固定取外側面深度，導致 `isOuter=false` 時孔被內側面蓋住。修正為取實際可見面（`isOuter ? mainOF : mainIF`）的深度。

### TD-03　正面視圖折疊段採正交前視縮短，不展開為平面圖

**決策：** 折疊段在正面視圖中以 `cos(foldRad)` 縮短顯示（D_u = FOLD_LEN × (1 − cos)），呈現真實的正交投影外觀。

**捨棄的替代方案：** 展開圖（unrolled）：不論折疊角，折疊段始終顯示完整 FOLD_LEN 高度。

**理由：** 展開圖無法隨角度更新，失去視覺意義；正交前視縮短可直觀呈現折疊程度，且實作僅需一個 scalar `cosFold`。

### TD-04　正面視圖頂部對齊側視圖 A 點，寬度標注移至板底

**決策：** `fvOY = margin + viewOY + bH_px`（頂部貼齊），板寬標注移至板底下方。

**捨棄的替代方案：** 正面視圖垂直置中（原始實作）；板寬標注在板頂上方。

**理由：** 兩視圖上緣齊平使比對更直觀；置中版在頂部留有不對稱空白。板頂置中後頂部 margin 可能不足以容納標注線，移至板底可保證空間。

## 5.2 `draw()` 執行順序（補充）

步驟 7（建立 drawList）新增螺絲孔 entry，插入於 ghost C 之前：

7a. **加入主板螺絲孔 entry**（`show3D && showScrewHoles` 時）：
   - `mainIsOuter` = sArea(外側面投影) < 0
   - depth = faceDepth(可見面) − 0.001
   - drawFn：繪製 u ≥ FOLD_LEN 的所有圓孔與格心點

7b. **加入折疊段螺絲孔 entry**（`show3D && showScrewHoles && showFold` 時）：
   - `foldIsOuter` = sArea(折疊外側面投影) < 0
   - depth = faceDepth(可見面) − 0.001
   - drawFn：繪製 u < FOLD_LEN 的所有圓孔與格心點
