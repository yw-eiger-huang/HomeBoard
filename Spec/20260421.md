# 木板構型幾何規格文件

> 版本：4.4 | 座標系：3D 斜平行投影（cabinet oblique）+ Y 軸旋轉 | 單位：公尺（除非特別標註）

---

## ── 基礎定義 ──

## 1. 基本構型描述

一塊木板以單點 hinge（A 點）固定於天花板，木板**外傾**（底端朝牆方向斜下），底端在設計角度（40°）時剛好接觸牆面。木板底部 65 cm 為可折疊段，以 hinge 繞 B 點旋轉，摺疊方向僅限朝牆面方向（往內折）。**摺疊角度無 UI 控制，隨牆面夾角自動同步。**

主板（A→B）與折疊段（B→D）以 3D 矩形柱體渲染；剛性桿（H-M-G）以圓柱三層描線表現，前後各一組；2D 模式只繪製 z=0 那側。

---

## 2. 座標系與邊界

| 元素 | 定義 |
|------|------|
| 原點 (0, 0, 0) | 牆面與地面交點（正面） |
| +X | 遠離牆面方向（水平） |
| +Y | 向上（垂直） |
| +Z | 朝觀察者方向（正面 = Z=0，背面 = Z=BOARD_W） |
| 牆面 | x = 0 |
| 地面 | y = 0 |
| 天花板 | y = CEIL_H = 2.500 m |

## 3. 固定參數

| 常數 | 值 | 說明 |
|------|----|------|
| `BOARD_LEN` | 3.048 m | 板全長（含摺疊段） |
| `FOLD_LEN` | 0.650 m | 摺疊段長（B→D） |
| `MAIN_LEN` | 2.398 m | 主板段長（A→B）= BOARD_LEN − FOLD_LEN |
| `T` | 0.018 m（1.8 cm） | 面板厚度（外側面） |
| `T_FRAME` | 0.045 m（4.5 cm） | 組合總深 = T + 0.027 m |
| `HT` | T_FRAME / 2 = 0.0225 m | 截面中心線偏移 |
| `BOARD_W` | 3.660 m | 板寬（Z 軸深度） |
| `CEIL_H` | 2.500 m | 天花板高 |
| `AX_W` | 1.9593 m | A 點離牆（= 3.048 × sin 40°） |

A、G、B 鉸接點位於外側面（距內側面 `T_FRAME` 處）。所有外側邊點（Ao、Bo、Co、Bfo、Do）以 `T_FRAME × 法向量` 從內側邊偏移。

---

## 4. 幾何關鍵點定義

所有坐標以**靠牆側邊緣（內側，w 字尾）**為基準（含 G、M）。A、H、I 為固定點；其餘點隨 θ 變動。

| 點 | 名稱 | 說明 |
|----|------|------|
| **Aw** | Hinge | 天花板固定點，靠牆側邊緣 |
| **Bw** | 折點 | 主板底端 = 摺疊段頂端，靠牆側邊緣 |
| **Cw** | 主板底端 | 木板全長末端（未折疊位置），靠牆側邊緣 |
| **Dw** | 摺疊端 | 摺疊段末端，靠牆側邊緣 |
| **Gw** | 剛性桿接點 | 主板靠牆側邊緣上，距 Aw 點 `g`（可調）之點 |
| **H** | 牆角 | 天花板與牆面交點 (0, 2.5)，固定 |
| **I** | 全長觸牆點 | θ=40° 時 Cw 的位置（觸牆參考點），固定 |
| **M** | 剛性桿交界點 | H-M、M-G 兩段等長剛性桿的交界點 |

## 5. 座標計算公式

### 5.1 輸入變數

| 變數 | 範圍 | 說明 |
|------|------|------|
| `θ`（angle） | 0° ~ 40° | 木板靠牆側邊緣與牆面的夾角；0° = 垂直貼牆，40° = 設計角度 |
| `g`（gOffset） | 1.75 ~ 2.15 m | Gw 點距 Aw 點沿板邊距離（使用者可調） |

### 5.2 方向向量

```
dir      = (−sin θ, −cos θ)              ← 主板方向（Aw→Bw）
norm     = (+cos θ, −sin θ)              ← 主板外法向量（遠離牆側）
boardDir = atan2(−cos θ, −sin θ)
```

### 5.3 靠牆側邊緣座標（inner edge）

```
Aw = (1.9593, 2.500)                                       ← 固定
Bw = Aw + MAIN_LEN × dir
Cw = Aw + BOARD_LEN × dir
Gw = Aw + g × dir
H  = (0, 2.500)                                            ← 固定
I  = (0, 2.500 − BOARD_LEN × cos 40°) ≈ (0, 0.165)       ← 固定
```

### 5.4 摺疊段（自動同步）

```
foldRad = π × (1 − θ/40°)               ← θ=40° 時 0°（D=C），θ=0° 時 180°
foldDir = boardDir − foldRad
fdx     = cos(foldDir)
fdy     = sin(foldDir)
fnx     = −fdy,  fny = fdx              ← 摺疊段外法向量

Dw = Bw + FOLD_LEN × (fdx, fdy)
```

### 5.5 外側邊緣（o 字尾）與中心線（c 字尾）

```
── 主板外側（+ norm × T_FRAME）──
Ao = Aw + T_FRAME × norm
Bo = Bw + T_FRAME × norm
Co = Cw + T_FRAME × norm

── 摺疊段外側（+ fold_norm × T_FRAME）──
Bfo = Bw + T_FRAME × (fnx, fny)
Do  = Dw + T_FRAME × (fnx, fny)

── 主板中心線（+ norm × HT）──
Ac = Aw + HT × norm
Bc = Bw + HT × norm
```

### 5.6 剛性桿交界點 M（兩等長桿，動態長度）

**桿長（隨 gOffset 動態計算）：**
```
LEN_HM = LEN_MG = 0.5 × √(AX_W² + g²)    ← 以 θ=0° 時 H→G 的一半為基準
```

**任意 θ 時：**
```
d    = |H − Gw|
a    = d / 2
h    = √(LEN_HM² − a²)
pm   = H + a × (Gw − H) / d              ← H→G 連線上的基點
perp = (−(Gw.y − H.y),  Gw.x − H.x) / d ← H→G 的左法向量
cross = (Gw − H) × (Aw − H)              ← z 分量：Aw 在 H→G 的哪一側

若 cross ≥ 0：M = pm + h × perp
否則：        M = pm − h × perp
```

> θ=0° 時 h=0，M = pm；θ 增大時 M 朝 Aw 方向彎折。

## 8. 輔助量測

| 量測項 | 公式 |
|--------|------|
| H→G 距離 | `\|H − Gw\|` |
| H→B 距離 | `\|H − Bw\|` |
| M→A 距離 | `\|M − Aw\|` |
| A→B 水平投影 | `Aw.x − Bw.x` |
| B 離地 | `Bw.y` |
| C 離地 | `Cw.y` |
| 內縮基準點離地高度 | 見 §8.1 |

### 8.1 內縮基準點投影高度

使用者可調整「內縮距離」d（100～150 cm），以 Ao 的地面投影為基準往牆面方向內縮 d，向上投影至主板外側面，標示交點離地高度。

```
refFloorX = Ao.x − d
t         = (refFloorX − Bo.x) / (Ao.x − Bo.x)    （t ∈ [0,1] 時有效）
projY     = Bo.y + t × (Ao.y − Bo.y)
```

`t ∉ [0,1]` 時標示自動隱藏。**視覺元素（`showDims` 開啟時）：** 地面水平 dim 線（refFloorX → Ao 地面投影）、垂直灰虛線（(refFloorX, 0) → (refFloorX, projY)）、交點灰色圓點、高度標籤（cm）。

## ── 3D 投影 ──

## 9. 投影規格

### 9.1 Cabinet Oblique 基本參數

```
投影角 30°，foreshortening 0.5×
OBL_COS = cos(30°) × 0.5 ≈ 0.4330
OBL_SIN = sin(30°) × 0.5 = 0.25
```

### 9.2 水平視角旋轉（Y 軸旋轉）

視角旋轉角 φ（`params.hViewAngle`，−90°～90°）繞 Y 軸旋轉整個場景：

```
cosV = cos(φ),  sinV = sin(φ)
h(x,z) = x·cosV + z·sinV          ← 水平螢幕分量
d(x,z) = −x·sinV + z·cosV         ← 深度分量

p(x, y, z):
  canvas_x = OX + h·S + d·S·OBL_COS
  canvas_y = OY − y·S − d·S·OBL_SIN
```

φ=0°：側視圖（h=x, d=z）。φ=±90°：板寬水平展開。水平視角滑桿僅在 3D 模式下顯示。**模式切換**：切換至 2D 時，φ 重設為 0° 並同步更新滑桿；切換回 3D 時，還原切換前的 φ 值（儲存於 `savedHViewAngle`）。

### 9.3 場景縮放（中心旋轉軸）

```
sceneW = AX_W + T_FRAME + 0.5
sceneH = CEIL_H × 1.22
S      = min((W−2·margin)/sceneW, (H−2·margin)/sceneH) × viewScale
```

OX 以場景 XZ 中心點 `(cx, cz) = (sceneW/2 − 0.25, BOARD_W/2)` 在 φ=0 時的投影水平座標為固定基準，旋轉時場景在畫布上原地旋轉不偏移：

```
hcCenter0 = cx + cz·OBL_COS
hcCenter  = (cx·cosV + cz·sinV) + (−cx·sinV + cz·cosV)·OBL_COS
OX        = margin + viewOX + 44·dpr + (hcCenter0 − hcCenter)·S
```

## 10. 板料構造（面板 + 邊框）

板子採「面板在前、邊框在後」構造：面板（`T` = 1.8 cm）覆蓋外側面（朝房間），無邊框外露；邊框（`T_FRAME − T` = 2.7 cm）固定於板面後方（內側）。邊框可見寬度由 `borderMarkWidth`/`borderMarkJunctionWidth` 控制（見 §16）。

## 12. 剛性桿繪製

3D 模式：**遠組**（depth 較大的 z 端）圓柱三層描線（body=#6aac9a，6×dpr）；**近組**（depth 較小的 z 端）標準色（body=#90d8c0，8×dpr）含長度標籤；深度連接線 `#90d8c055`，1.5px；遠近判斷 `cosV ≥ 0` → z=BW 較遠。2D 模式：只繪近組（z=0），無連接線。

### 12.1 圓柱三層描線規格

```
shadow：    strokeStyle=暗色，lineWidth=2×dpr，偏移 +1.5px（法向量方向）
body：      strokeStyle=主色，lineWidth=8×dpr（近）/ 6×dpr（遠）
highlight： strokeStyle=亮色，lineWidth=2.5×dpr，偏移 −1.5px
lineCap = 'round'
```

## ── 邊框與標示 ──

## 16. 邊框標示（Border Mark）

在主板（A→B）與折疊板（B→D）四邊往內標示填色安全邊距。`showBorderMark`（預設 on）關閉時兩個寬度 slider row 一併 `display: none`。

### 16.1 邊框寬度參數

| 滑桿 ID | 參數鍵 | 範圍 | 預設 | 作用邊 |
|---------|--------|------|------|--------|
| `#borderMarkWidth` | `borderMarkWidth` | 3.6–10 cm | 5 cm | A 端、D 端、左右 Z 邊 |
| `#borderMarkJunctionWidth` | `borderMarkJunctionWidth` | 3–5 cm | 3 cm | B 交接端（主板 B 端 + 摺疊板 B 端） |

Step 均為 0.1 cm；值以公尺儲存（`cm / 100`）。

### 16.2 幾何定義

設 `M = borderMarkWidth`，`J = borderMarkJunctionWidth`。

**正面視圖（Y-Z 投影，`frontview.js`）：**

主板（MAIN_LEN × BOARD_W）：
```
內框上邊  = AY_w − M × cos(θ)     // A 端往內 M
內框下邊  = BY_w + J × cos(θ)     // B 端往內 J
內框左邊  z = M
內框右邊  z = BOARD_W − M
```

摺疊板（FOLD_LEN × BOARD_W）：
```
內框上邊  = BY_w + J × fdy         // B 端往內 J
內框下邊  = DY_w − M × fdy         // D 端往內 M
內框左邊  z = M
內框右邊  z = BOARD_W − M
```

**3D 視圖（`draw.js`）：** 在可見大面（shoelace 法判斷）繪製外框 minus 內框（`ctx.fill('evenodd')`）；topMarg/botMarg 對應兩端各自邊距（主板：top=M, bot=J；摺疊板：top=J, bot=M）；Z 方向固定使用 M。

**側視圖 2D（`draw.js`，`show3D=false`）：** 在板端截面繪製填色四邊形；主板 A 端/B 端各寬 M/J，摺疊板 B 端/D 端各寬 J/M。

### 16.3 視覺樣式

| 屬性 | 值 |
|------|-----|
| 填色 | `rgba(0,0,0,0.35)` |
| 繪製方式 | `ctx.fill('evenodd')` |

## 17. 雙層厚度標註（側視圖，`showDims = true`）

### 17.1 B 點標註

## 19. 2D 正面視圖

`show3D = false` 時，canvas 分左右兩欄：左欄側視圖，右欄正面視圖（`drawFrontView()`），分隔線虛線 `#3a4060` 於 `x = W/2`。

### 19.1 投影方式

正交投影，觀察方向沿 **+X 軸**（從室內看向牆面），投影面為 Y-Z 平面。

| 軸 | 含意 | 範圍 |
|----|------|------|
| 橫軸 Z | 板寬 | 0 → BOARD_W |
| 縱軸 Y | 離地高度 | 0 → CEIL_H |

Canvas 座標函式（與側視圖共用比例尺 `S`）：
```
cvX(z) = fvOX + z·S
cvY(y) = margin + viewOY + (CEIL_H − y)·S
fvOX   = W/2 + (W/2 − margin − BOARD_W·S) / 2     ← 板寬置中於右欄
```

### 19.2 關鍵高度

```
AY_w = CEIL_H
BY_w = AY_w − MAIN_LEN × cos(θ)
fdy  = −cos(θ + foldRad)
DY_w = BY_w + FOLD_LEN × fdy
```

### 19.3 圓孔投影位置

| 區段 | 公式 |
|------|------|
| 主板孔（`u ≥ FOLD_LEN`） | `hy = AY_w − (BOARD_LEN − u) × cos(θ)` |
| 折疊段孔（`u < FOLD_LEN`） | `hy = BY_w + (FOLD_LEN − u) × fdy` |

開關同 §13.5；`showFold = false` 時折疊段孔不繪製。

### 19.4 標注與標籤

| 元素 | 說明 |
|------|------|
| 板寬標注 | `BOARD_W = 3.66m`，水平標注線位於最低邊緣下方 `dimOff` |
| 主板高標注 | `MAIN_LEN`，縱向標注線於板右側 `dimOff` |
| 折疊高標注 | `FOLD_LEN`，縱向標注線於板右側 `dimOff×2`；投影高度 ≤ 5px 時隱藏 |
| 點標籤 | A（板頂中央）、B（左邊緣折線處）、D（折疊段末端中央） |
| 段落標籤 | `A→B 2.40m`（主板中央）、`B→D 0.65m`（折疊段中央） |
| 標題 | `正面視圖`（canvas 底部右欄）、`側視圖`（canvas 底部左欄） |

### 19.5 比例尺與縮放

側視圖與正面視圖共用同一比例尺 `S = autoS × viewScale`。2D 模式下：
```
sideW  = W/2 − margin − 44·dpr
frontW = W/2 − margin − 90·dpr    ← 右側留給標注線的空間
autoS  = min(sideW/sceneW, frontW/BOARD_W, (H − margin×2)/sceneH)
```

## 20. 摺疊板折疊範圍與隱藏規則

### 20.1 折疊角公式

`foldRad = π × (1 − angle/40)`

| 牆面夾角 θ | foldRad | 說明 |
|-----------|---------|------|
| 0° | π（180°） | 完全折回，與主板平行反向 |
| 40° | 0° | 與主板共面（Dw = Cw） |

### 20.2 正面視圖隱藏條件

`fdy = sin(foldDir) = −cos(θ + foldRad)`

當 `fdy >= 0`（B→D 朝天花板方向），摺疊板已折回主板背面，**僅正面視圖**隱藏摺疊板相關元素；側視圖不受影響（見 §20.3）。

臨界角：`θ < 180/7 ≈ 25.7°` 時 `fdy > 0`（隱藏）；`θ ≥ 25.7°` 時顯示。

隱藏元素（`fdy >= 0` 時，`frontview.js`）：摺疊板矩形、邊框標示、螺絲孔/格點、分割線、`B→D` 標籤、D 點標籤、尺寸線。

### 20.3 側視圖恆顯示

側視圖（X-Y 投影）中，摺疊板沿 fdx（恆負，朝牆方向）從 B 延伸，永遠不被主板遮擋。`show3D = false` 時無條件繪製所有摺疊板元素（前面 z=0、邊框標示、D 關鍵點、`B→D` 標籤）。`show3D = true` 由深度排序自然處理。

## 25. 響應式介面（RWD）

### 25.1 版面邏輯

| 情境 | 條件 | 面板位置 | 面板寬度 |
|------|------|----------|----------|
| 桌面版 | viewport 寬 > 640px | 右側 | 240px |
| 手機直式 | `max-width: 640px` + portrait | 畫布下方 | 全寬 |
| 手機橫式 | `max-height: 500px` + landscape | 右側（緊湊）| 180px |

手機版字體縮減：label 11px、stat 10px。直式：主佈局垂直 flex，面板 `width:100%` + `border-top`，section 水平 flex wrap。橫式：沿用水平 flex，section padding 縮減為 8px 10px。

### 25.2 Canvas 繪圖縮放與定位

```
margin = min(90×dpr, min(W,H) × 0.025)
sceneW = AX_W + T_FRAME + 0.5
sceneH = CEIL_H × 1.22
autoS  = min((W − 2×margin) / sceneW, (H − 2×margin) / sceneH)
S      = autoS × viewScale

OX = margin + viewOX + 44×dpr   ← 牆面左側 annotation 留白
OY = margin + CEIL_H×S + viewOY ← 天花板錨定於頂部 margin
```

初始（viewScale=1）場景填滿 canvas 短邊的 95%；天花板線始終錨定於頂部 margin。

觸控：單指拖曳平移，雙指捏合縮放（0.15～12×），所有 touch 事件 `preventDefault()`（`passive: false`）；hint 文字 `拖曳平移 · 滾輪縮放 · 雙指縮放`。

## 26. 程式碼模組結構

`board_viz.js` 依職責拆分為 ES Modules（`type="module"`）：

| 檔案 | 職責 |
|------|------|
| `constants.js` | 所有物理/視覺常數與網格陣列（`BOARD_LEN`、`FS`、`SCREW_Z` 等） |
| `geometry.js` | `getGeom()`、`clampAngle()` |
| `state.js` | `canvas`、`ctx`、`params`、`view`（含平移/縮放/拖曳狀態） |
| `frontview.js` | `drawFrontView()` 及內部標注輔助函式 |
| `draw.js` | `draw()`、`drawFace3D()`、點標記輔助函式 |
| `events.js` | 所有滑桿、切換按鈕、滑鼠、觸控、視窗 resize 監聽器；`resize()` |
| `board_viz.js` | 入口點：`import { resize } from './events.js'; resize();` |

模組依賴方向（無循環）：`board_viz.js` → `events.js` → `draw.js` → `frontview.js` → `state.js`/`constants.js`；`geometry.js` → `state.js`/`constants.js`。

## 27. Git Hooks

Hook 腳本置於 `.githooks/`（納入版本控制），以 `git config core.hooksPath .githooks` 啟用。Clone 後需執行一次此指令。

### 27.1 `pre-commit`（封鎖）

| 規則 | 觸發條件 | 訊息前綴 |
|------|---------|---------|
| R1 | `Code/` 有 staged 檔案，但無任何 `Spec/` staged 檔案 | `[R1]` |
| R2 | `Spec/YYYYMMDD.md` staged，但對應 `Spec/YYYYMMDD.design.md` 未 staged | `[R2]` |
| R3 | 新建 spec 檔案（`--diff-filter=A`）未與其對應檔案同時出現 | `[R3]` |
| R4 | 同時 staged 的 `YYYYMMDD.md` 與 `YYYYMMDD.design.md` 版本號不一致 | `[R4]` |

任一規則命中 → `exit 1` 中止 commit。

> git 無 pre-checkout hook，分支切換規則由 `CLAUDE.md` 強制：**`Code/` 有未提交異動時，禁止切換分支；必須先 commit 或 stash。**
